# isagro-barra-horizontal

## Política de backup

A política de backup deste projeto deve ser definida pelo cliente responsável pelo ambiente.

O cliente deve informar e manter documentado:

- tipo de storage utilizado;
- capacidade disponível e crescimento esperado;
- ambiente de armazenamento dos backups;
- dados, bancos, arquivos e volumes incluídos no backup;
- dados, bancos, arquivos e volumes excluídos do backup;
- frequência de execução;
- janela de execução;
- período de retenção;
- objetivo de ponto de recuperação;
- objetivo de tempo de recuperação;
- responsáveis pela operação;
- responsáveis pela validação;
- procedimento de restauração;
- frequência de teste de restauração;
- requisitos de acesso, segurança e conformidade.

Sem essas definições, o projeto não deve assumir uma estratégia de backup. Qualquer plano de backup deve estar documentado no README do projeto ou em documento operacional referenciado pelo README.
